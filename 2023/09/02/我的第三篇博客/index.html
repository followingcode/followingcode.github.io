问题描述：

两个非递减的整数数组，num1和num2，而num1和num2中数组元素的个数是m和n

合并两个数组，使合并之后的数组仍然按非递减顺序排列

注意：合并之后的数组存储在num1中，为了应对这种情况，nums1 的初始长度为 m + n，其中前 m 个元素表示应合并的元素，后 n 个元素为 0 ，应忽略。nums2 的长度为 n 。

解决思路：方法1

比较容易想到的直观的思路是直接将数组num2插入到数组num1尾部，然后进行排序

nums2[0]赋值给了nums1[m],nums2[1]赋值给了nums1[m+1]，……..nums2[n-1]赋值给了nums1[m+n-1],刚好装满数组nums1，然后对数组nums1中的元素进行排序。

方法2

双指针来思考，将两个数组看作队列，每次从两个数组头部取出比较小的数字放到结果中。

我们为两个数组分别设置一个指针p1与 p2 来作为队列的头部指针。代码实现如下：

分析：数组nums1或数组nums2没有遍历完，循环就会一直执行下去，比如说数组nums1数组长度本身比较小，遍历很快结束，于是p1<m条件不满足，但p2<n条件满足，于是p1<m||p2<n条件满足，但如果换成p1<m&&p2<n，条件就会不满足，while循环就不会执行大花括号里面的语句，这样导致的后果就是循环遍历的元素没有遍历完漏掉了待排序元素。

当p1指针指向数组nums1的最后一个元素nums1[m-1],继续后移，p1指针指向数组nums1元素nums1[m]，

//p1到头了

//当前要添加的是p2下一个元素，并且p2更新

cur = nums2[p2];

p2++;

cur可以简单理解为临时变量temp，将p2指针指向的数组元素值存储在临时变量里面

然后p2更新后移

else表示上一个条件不满足的情况下，即p1没有到头的情况下并且p2到头这种情况下

如果p2到头了，当前要添加的是p1下一个元素，并且p1更新，

cur可以简单理解为临时变量temp，将p1指针指向的数组元素值存储在临时变量里面

然后p1更新后移

四种情况，初始情况p1指针和p2指针都是指向数组第一个元素

p1到头，p2没有到头， if (p1 == m)，这种情况不需要再判断到底是谁小谁大比较之后再放结果数组这种了，只需要将p2指向的元素放入结果数组，再更新后移指针即可

p1没有到头，p2到头，先else否定上一个条件再 if (p2 == n)，这种情况不需要再判断到底是谁小谁大比较之后再放结果数组这种了，只需要将p1指向的元素放入结果数组，再更新后移指针即可

p1没有到头而且p2没有到头

将哪个元素放入结果数组，是p1指向的元素还是p2指向的元素？肯定是小的先放

p1到头，p2到头，

if条件判断满足，执行语句cur = nums2[p2++];接着执行语句sorted[p1 + p2 - 1] = cur;

if条件判断不满足，不会执行语句cur = nums2[p2++];而是跳转到else if

不妨测试样例假设1 2 3，于是m=1，n=2

p1=0，满足p1<1||p2<2,于是

即cur=1，p1=p1+1=1，则p1+p2-1=1+0-1=0，sorted[p1 + p2 - 1] = cur;辅助空间数组

重新回到while循环，1<1||0<2条件满足

即cur=2，p2=p2+1=1，则p1+p2-1=1+1-1=1，sorted[p1 + p2 - 1] = cur;辅助空间数组

重新回到while循环，1<1||1<2条件满足

即cur=3，p2=p2+1=2，则p1+p2-1=1+2-1=2，sorted[p1 + p2 - 1] = cur;辅助空间数组

重新回到while循环，1<1||2<2条件不满足，跳出while循环

分析：nums1[0]=1,i=1满足i !=3,

nums1[1]=2,i=2满足i !=3,

nums1[2]=3,i=3不满足i !=3,于是跳出来for循环，循环结束
