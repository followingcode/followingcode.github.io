GitHub 术语解释

Repository：简称Repo，可以理解为“仓库”，我们的项目就存放在仓库之中。也就是说，如果我们想要建立项目，就得先建立仓库；有多个项目，就建立多个仓库。

Issues：可以理解为“问题”，举一个简单的例子，如果我们开源一个项目，如果别人看了我们的项目，并且发现了bug，或者感觉那个地方有待改进，他就可以给我们提出Issue，等我们把Issues解决之后，就可以把这些Issues关闭；反之，我们也可以给他人提出Issue。

Star：可以理解为“点赞”，当我们感觉某一个项目做的比较好之后，就可以为这个项目点赞，而且我们点赞过的项目，都会保存到我们的Star之中，方便我们随时查看。在 GitHub 之中，如果一个项目的点星数能够超百，那么说明这个项目已经很不错了。

Fork：可以理解为“拉分支”，如果我们对某一个项目比较感兴趣，并且想在此基础之上开发新的功能，这时我们就可以Fork这个项目，这表示复制一个完成相同的项目到我们的 GitHub 账号之中，而且独立于原项目。之后，我们就可以在自己复制的项目中进行开发了。

Pull Request：可以理解为“提交请求”，此功能是建立在Fork之上的，如果我们Fork了一个项目，对其进行了修改，而且感觉修改的还不错，我们就可以对原项目的拥有者提出一个Pull请求，等其对我们的请求审核，并且通过审核之后，就可以把我们修改过的内容合并到原项目之中，这时我们就成了该项目的贡献者。

Merge：可以理解为“合并”，如果别人Fork了我们的项目，对其进行了修改，并且提出了Pull请求，这时我们就可以对这个Pull请求进行审核。如果这个Pull请求的内容满足我们的要求，并且跟我们原有的项目没有冲突的话，就可以将其合并到我们的项目之中。当然，是否进行合并，由我们决定。

Watch：可以理解为“观察”，如果我们Watch了一个项目，之后，如果这个项目有了任何更新，我们都会在第一时候收到该项目的更新通知。

GAN

同时训练两个模型，生成模型G，判别模型D

判别模型D：估计样本来自训练数据而不是来自生成模型G的概率

生成模型G：使判别模型D出错的概率最大化

在这篇论文中，这个解是指GAN模型中的最优解，其中G和D是两个对抗性的神经网络模型。这个解的存在保证了GAN模型的训练能够找到一个最佳的生成模型和判别模型，以此来生成高质量的样本。

在 G 和 D 由多层感知器定义的情况下，整个系统可以使用反向传播进行训练。

分析思路：

The promise of deep learning is to discover rich, hierarchical models [2] that represent probability distributions over the kinds of data encountered in artificial intelligence applications, such as natural images, audio waveforms containing speech, and symbols in natural language corpora

分析：通俗地说，这句话的意思是，深度学习的目标是构建一种能够理解和处理各种不同类型的数据的系统。这些数据可以是自然图像、语音录音中的声音、或者自然语言文本。深度学习的核心思想是通过构建多层次的模型来实现这一目标，就像构建一个大型的数据处理和理解系统一样。

这些模型通过逐渐提取数据中的不同特点和模式，将数据分层次地表示为概率分布。这有助于计算机更好地理解数据，从而能够执行各种任务，比如识别图像中的物体、转录语音成文字、或者理解自然语言的含义。深度学习已经在许多领域取得了显著的进展，包括图像识别、语音识别、自然语言处理和许多其他人工智能应用。

分层模型是指一种模型或系统，它由多个层次或层次结构组成，每个层次都有不同的功能和任务。在深度学习中，这些层次通常表示神经网络的不同层。

在深度学习中，分层模型是指多层神经网络，每一层都包含一组神经元或节点。这些层次通常分为输入层、隐藏层和输出层，每个层次都执行不同的计算任务。数据从输入层传递到输出层，通过多个隐藏层进行处理。每个隐藏层都学会识别数据中的不同特征和模式，从而构建了一种分层次的表示。

分层模型的关键思想是通过多次处理数据来逐步提取数据的高级特征。这有助于模型更好地理解和表示复杂的数据，例如图像、语音或文本。深度学习中的分层模型通常采用反向传播算法进行训练，以调整模型的权重和参数，以便使其能够更好地执行特定任务，如图像分类、语音识别或自然语言处理。这种分层的方法已经在深度学习中取得了很大的成功。

So far, the most striking successes in deep learning have involved discriminative models, usually those that map a high-dimensional, rich sensory input to a class label [14, 20].

迄今为止，在深度学习中最令人瞩目的成就通常涉及一类模型，即判别模型。这些模型的主要任务是将复杂的、高维度的感知数据（比如图像、声音等）映射到类别标签或类别预测上。判别模型的目标是找到输入数据与特定类别之间的关联，以便进行有效的分类或标识。这种方法已经在多个领域取得了成功，例如图像分类、语音识别等。

总之，这句话强调了深度学习中判别模型在高维感知数据处理和类别预测方面的卓越成就。这些模型已经被广泛用于解决各种问题，特别是在涉及大量数据和复杂模式的任务中。

反向传播（Backpropagation）之所以被称为”反向”传播，是因为它是一种从输出层到输入层的误差信息传递的过程，与前向传播（Forward Propagation）相反。前向传播是神经网络中数据从输入层流向输出层的过程，计算神经网络的输出值。而反向传播则是在计算输出后，将误差信号从输出层逆向传递回神经网络的每一层，以便进行权重和参数的调整，以最小化模型的误差。

具体来说，反向传播通过计算预测值与实际值之间的误差，并沿着神经网络的层次结构，逆向计算每一层的梯度（即误差信号），以确定每个层的权重应该如何进行微小的调整，以减小误差。这个过程的核心思想是将误差信息从输出层逐渐传播回到网络的输入层，以便更新网络参数以提高预测准确性。

所以，”反向传播”之所以被命名为反向，是因为它是信息的反向传递，从输出层到输入层，以进行权重调整和网络训练的过程。这一过程是深度学习训练中非常重要的一部分，因为它使神经网络能够逐渐学习如何更好地拟合训练数据。

在数学中，梯度是一个向量，它表示一个函数在某一点上的变化率和方向。梯度通常用符号∇（nabla）表示，后面紧跟着函数的名称，如∇f，表示函数f的梯度。梯度是一个多维向量，其每个分量分别表示函数在对应坐标轴上的偏导数。

具体而言，对于一个函数f(x₁, x₂, …, xₙ)，其梯度∇f 是一个由偏导数组成的向量，表示为 (∂f/∂x₁, ∂f/∂x₂, …, ∂f/∂xₙ)。这个向量告诉我们在函数的每个输入变量x₁, x₂, …, xₙ上，函数值增加最快的方向和速度。
